## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `gray-900` (dark) | Page background, card surfaces |
| Secondary (30%) | `gray-50` / `gray-800` (dark) | Group header backgrounds, hover states |
| Accent (10%) | `#3b82f6` (accent-500 / blue-500) | Links, highlight ring, back-navigation text |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for: anchor link highlights, navigation links, "back" link, total question count text. NOT used for type badges (badges use their own semantic colors below).

### Type Badge Colors

Each question type has a dedicated color pair (background + text). These are NOT accent colors -- they are semantic category indicators.

| Type | Light mode | Dark mode | Tailwind classes |
|------|-----------|-----------|------------------|
| Deep | bg: `#dbeafe` (blue-100), text: `#1e40af` (blue-800) | bg: `#1e3a5f` (blue-900/50), text: `#93c5fd` (blue-300) | `bg-blue-100 text-blue-800 dark:bg-blue-900/50 dark:text-blue-300` |
| Trick | bg: `#fef3c7` (amber-100), text: `#92400e` (amber-800) | bg: `#78350f/50` (amber-900/50), text: `#fcd34d` (amber-300) | `bg-amber-100 text-amber-800 dark:bg-amber-900/50 dark:text-amber-300` |
| Practical | bg: `#d1fae5` (green-100), text: `#065f46` (green-800) | bg: `#064e3b/50` (green-900/50), text: `#6ee7b7` (green-300) | `bg-green-100 text-green-800 dark:bg-green-900/50 dark:text-green-300` |
| Basic | no badge displayed | no badge displayed | n/a |

## Component Inventory

### New: TypeBadge

A small pill rendered inline on the flashcard summary row. Only shown when `type !== 'basic'`.

**Structure:**
```html
<span class="inline-flex items-center rounded-full px-2 py-0.5 text-xs font-semibold {color-classes}">
  {label}
</span>
```

**Placement:** Inside the `<summary>` element of `Flashcard.astro`, between the question text `<span>` and the anchor link `<a>`. Vertically centered via the existing `flex items-center` on summary.

**Labels (bilingual):**
| Type | EN label | UA label |
|------|----------|----------|
| deep | Deep | Глибоке |
| trick | Trick | Пастка |
| practical | Practical | Практичне |

**Props:**
```typescript
interface Props {
  type: 'deep' | 'trick' | 'practical';
  locale: 'ua' | 'en';
}
```

### Modified: Flashcard.astro

**Changes:**
- Accept new optional prop: `type?: 'basic' | 'deep' | 'trick' | 'practical'` (defaults to `'basic'`)
- Accept new prop: `locale: 'ua' | 'en'`
- Render `TypeBadge` inside `<summary>` when `type !== 'basic'`
- Badge appears as a `shrink-0` element so it never wraps to a new line

### Modified: FlashcardList.astro

**Changes:**
- Accept `locale` prop and pass it through to each `Flashcard`
- Accept `type` in the question data interface and pass it through

### New: TypeGroup section on [section]/index.astro

**Changes to section page:**
- Group questions by type in order: Basic, Deep, Trick, Practical
- Each non-empty group gets a group header
- Type count summary shown below section heading

**Group header structure:**
```html
<div class="flex items-center gap-3 pt-8 pb-3 border-b border-gray-200 dark:border-gray-700">
  <h2 class="text-lg font-semibold text-gray-900 dark:text-gray-100">{group label}</h2>
  <span class="text-sm text-gray-500 dark:text-gray-400">({count})</span>
</div>
```

**Group labels (bilingual):**
| Type | EN | UA |
|------|-----|-----|
| basic | Basic | Базові |
| deep | Deep Technical | Глибокі технічні |
| trick | Trick Questions | Питання-пастки |
| practical | Practical Scenarios | Практичні задачі |

**Type count summary (below section heading):**
Replaces the current single count line. Renders as a horizontal list of type counts using badge colors.

```html
<div class="flex flex-wrap items-center gap-3 mt-2">
  <span class="text-sm text-gray-600 dark:text-gray-400">{total} questions</span>
  <!-- Only rendered for non-zero types -->
  <span class="inline-flex items-center gap-1 text-xs font-medium text-blue-700 dark:text-blue-300">
    <span class="w-2 h-2 rounded-full bg-blue-500"></span> {n} Deep
  </span>
  <span class="inline-flex items-center gap-1 text-xs font-medium text-amber-700 dark:text-amber-300">
    <span class="w-2 h-2 rounded-full bg-amber-500"></span> {n} Trick
  </span>
  <span class="inline-flex items-center gap-1 text-xs font-medium text-green-700 dark:text-green-300">
    <span class="w-2 h-2 rounded-full bg-green-500"></span> {n} Practical
  </span>
</div>
```

## Interaction States

### TypeBadge
- **Default:** Pill with background color and text color per type
- **Hover:** No hover state (badge is non-interactive, purely informational)
- **Focus:** Not focusable (span element, not a link or button)

### Group Headers
- **Default:** Text heading with count, bottom border separator
- **Hover:** No hover state (non-interactive heading)
- **First group:** No top padding (`pt-0` instead of `pt-8` for the first group to avoid double spacing)

### Backward Compatibility
- Existing 105 "basic" questions render exactly as before (no badge, no visual change)
- The type field defaults to `'basic'` in the schema, so no existing content files need updating
