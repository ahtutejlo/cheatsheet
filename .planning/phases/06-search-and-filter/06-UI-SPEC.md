### Search Icon in Header

Position: Between locale switcher link and theme toggle button, inside the `div.flex.items-center.gap-1` container.

```
[site title]                    [EN/UA] [search icon] [theme toggle]
```

Icon: Magnifying glass, 20x20 (w-5 h-5), same styling as theme toggle button: `rounded-lg p-2 text-gray-500 hover:bg-gray-100 dark:text-gray-400 dark:hover:bg-gray-700 transition-colors`.

### Tag Badges on Flashcard

Position: Below the question text in the summary element, after the TypeBadge (if present) and before the anchor link icon.

Layout: Tags wrap in a flex row with gap-1, rendered only when tags array is non-empty.

```
[question text]  [TypeBadge?]  [tag1] [tag2] [tag3]  [anchor link] [chevron]
```

On small screens (< sm), tags should NOT appear in the summary to avoid crowding. Tags appear in a separate row below the summary on mobile or only within the expanded answer area. Recommended approach: render tags below the summary line on a second row within the summary element, visible at all breakpoints.

Revised layout:
```
Row 1: [question text]                    [TypeBadge?] [anchor] [chevron]
Row 2: [tag1] [tag2] [tag3]              (only if tags exist)
```

### Tag Filter Bar on Section Page

Position: Below the type count summary, above the first type group. Visible only when a tag filter is active.

```
[Section header]
[type counts: 15 Basic  5 Deep  5 Trick  5 Practical]
[Filtered by: {tag}  |  Show all]          <-- appears only when filter active
[type groups with flashcards]
```

### Clear Filter Button

Style: Text button, blue-600 text, no background. Appears inline after the active tag indicator.

  Show all

## Interaction Contracts

### Search (SRCH-01)

1. **Entry point:** User clicks search icon in header.
2. **Navigation:** Navigates to `/{locale}/search` page.
3. **Search input:** Auto-focused on page load. Pagefind handles debouncing (300ms default).
4. **Results:** Pagefind default UI renders results with highlighted matching text. Results link to section pages.
5. **No results:** Pagefind default UI shows "No results" message.
6. **Back navigation:** Browser back button returns to previous page.
7. **Dev mode:** Search index not available in dev mode. Pagefind shows placeholder message. Test with `npm run build && npm run preview`.

### Tag Filtering (SRCH-02)

1. **Click tag:** Clicking a tag badge on any flashcard filters the current section page to show only flashcards containing that tag.
2. **Visual feedback:** Clicked tag transitions to active state (blue-500 bg, white text). All other tag badges with the same tag value also highlight. Filter bar appears below type counts showing "Filtered by: {tag} | Show all".
3. **Hidden flashcards:** Flashcards without the selected tag get `display: none`. Empty type groups (where all flashcards are hidden) also get `display: none`.
4. **Clear filter - method 1:** Click the same active tag again to toggle off. All flashcards and type groups become visible again.
5. **Clear filter - method 2:** Click "Show all" button in the filter bar. Same effect as method 1.
6. **Switch tag:** Clicking a different tag while one is active replaces the filter (does not stack/AND tags).
7. **No URL state:** Tag filter is ephemeral client-side state. Refreshing the page clears the filter.

### Pagefind Indexing

1. **Indexed content:** Only content inside `<main data-pagefind-body>` on section pages.
2. **Excluded content:** Header navigation, "Back to sections" link, type group headers, type count summary. Use `data-pagefind-ignore` on these elements.
3. **Multilingual:** Automatic via `<html lang={locale}>`. UA pages indexed separately from EN pages.
