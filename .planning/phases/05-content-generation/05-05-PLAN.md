**TRICK questions (order 21-25, type: "trick"):**
Each answer MUST start with `> **Trap:** ...`:
1. `copy-vs-add-trap.md` (order 21) — Trap: ADD and COPY are identical in Dockerfile. Reality: ADD auto-extracts tar archives and supports URLs; COPY is simple file copy. Best practice: always use COPY unless you need extraction. Tags: [dockerfile, pitfalls, best-practices]
2. `entrypoint-cmd-interaction-trap.md` (order 22) — Trap: ENTRYPOINT and CMD are alternative ways to set the startup command. Reality: they work together -- ENTRYPOINT is the executable, CMD provides default arguments; CMD is overridden by `docker run` args, ENTRYPOINT is not (without --entrypoint). Tags: [dockerfile, entrypoint, cmd]
3. `docker-stop-sigkill-trap.md` (order 23) — Trap: `docker stop` sends SIGKILL immediately. Reality: sends SIGTERM first, waits grace period (default 10s), then SIGKILL. App must handle SIGTERM for graceful shutdown. Tags: [lifecycle, signals, graceful-shutdown]
4. `volume-mount-permissions-trap.md` (order 24) — Trap: bind mounts always have correct permissions inside the container. Reality: UID/GID mismatch between host and container causes permission denied; must align user IDs or use named volumes. Tags: [volumes, permissions, security]
5. `dockerignore-gitignore-trap.md` (order 25) — Trap: .dockerignore uses the same syntax as .gitignore. Reality: while similar, .dockerignore has differences -- no negation with `!` combining, context-relative paths, and it affects build context size (not just tracking). Tags: [dockerfile, build-context, pitfalls]
