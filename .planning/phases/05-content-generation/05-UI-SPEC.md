# Phase 5 — UI Design Contract

> Visual and interaction contract for the content generation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase context:** Phase 5 creates 90 Markdown content files. No new components, layouts, or styles are introduced. The UI contract for this phase defines how authored Markdown content must be structured so it renders correctly within the existing Flashcard component, TypeBadge component, and Tailwind Typography prose styling established in Phases 1-4.

### Content Authoring Patterns (Phase-Specific Contract)

These patterns define how authored Markdown renders within the existing `prose` container.

**Deep questions -- answer structure:**
```
[Core concept - 1 paragraph, 2-4 sentences]

[Implementation details / internal mechanics - 1-2 paragraphs]

```language
// Code showing internal mechanism or advanced usage
```

[Why it matters for interviews - 1-2 sentences]
```

**Trick questions -- answer structure:**
```
> **Trap:** {common misconception stated as a false claim, single sentence}

[Correct answer - 1-2 paragraphs explaining the truth]

```language
// Code demonstrating trap vs correct behavior
```

[Why this misconception is common - 1-2 sentences]
```

The `> **Trap:**` blockquote renders as a styled blockquote via Tailwind Typography. The bold "Trap:" label is the visual anchor that signals this is a trick question. Every trick question MUST begin with this exact pattern.

**Practical questions -- answer structure:**
```
**Scenario:** [realistic situation description, 1-2 sentences]

**Approach:**
1. [step 1]
2. [step 2]
3. [step 3]

**Solution:**
```language
// Concrete code or configuration
```
```

Bold labels (**Scenario:**, **Approach:**, **Solution:**) render as `<strong>` within prose and serve as structural anchors. All three sections are mandatory.

### Code Block Language Tags

Every fenced code block must specify a language tag for Shiki syntax highlighting:

| Section | Expected Languages |
|---------|--------------------|
| Automation QA | `java`, `python`, `bash`, `yaml` |
| Java | `java` |
| Kubernetes | `yaml`, `bash` |
| Blockchain | `solidity`, `javascript`, `bash` |
| Docker | `dockerfile`, `bash`, `yaml` |
| SQL | `sql`, `bash` |

Code blocks without a language tag render as plain text with no highlighting -- this is a visual defect.

### Bilingual Content Rules

- Code blocks are identical in UA and EN versions (never translate variable names, comments, or class names)
- Technical terms (Docker, JVM, SQL, Kubernetes, etc.) remain in English in Ukrainian text
- Ukrainian text uses natural phrasing, not word-for-word translation from English
- Both `ua_question` and `en_question` values must be wrapped in double quotes in YAML frontmatter
