2. `group-by-non-aggregated-trap.md` (order 22) — Trap: selecting non-aggregated columns with GROUP BY always returns an error. Reality: MySQL with ONLY_FULL_GROUP_BY disabled silently picks arbitrary values; PostgreSQL/standard SQL rejects it. Tags: [group-by, aggregations, sql-standard]

4. `correlated-subquery-performance-trap.md` (order 24) — Trap: correlated subqueries are always slower than JOINs. Reality: modern optimizers can transform correlated subqueries into joins; sometimes EXISTS with correlated subquery outperforms JOIN for semi-joins. Tags: [subqueries, performance, optimization]
5. `distinct-vs-group-by-trap.md` (order 25) — Trap: DISTINCT and GROUP BY (without aggregation) produce different results. Reality: they produce identical result sets for deduplication; GROUP BY is preferred when aggregation is also needed. Tags: [distinct, group-by, equivalence]
