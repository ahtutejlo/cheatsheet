1. Create `src/lib/markdown.ts` -- build-time Markdown-to-HTML renderer for frontmatter answer strings:
   ```typescript
   import { marked } from 'marked';

   // Configure marked for safe, consistent rendering
   marked.setOptions({
     gfm: true,
     breaks: false,
   });

   /**
    * Renders a Markdown string to HTML.
    * Used for frontmatter answer fields (ua_answer, en_answer)
    * that cannot use Astro's native render() API.
    *
    * Note: This does NOT include Shiki syntax highlighting.
    * Code blocks get <pre><code class="language-xxx"> which can be
    * styled with CSS. For full Shiki highlighting of frontmatter
    * Markdown, a remark/rehype pipeline would be needed -- deferred
    * to Phase 2 if the basic code styling is insufficient.
    *
    * Alternative: Use Astro's internal Markdown processor if available.
    */
   export async function renderMarkdown(content: string): Promise<string> {
     return await marked.parse(content);
   }
   ```

   IMPORTANT consideration: `marked` will produce `<pre><code class="language-java">` blocks but without Shiki token-level coloring. For Phase 1, this is acceptable -- code blocks will be styled with CSS (monospace font, background color, padding). Full Shiki highlighting for frontmatter strings would require importing Shiki directly and calling `codeToHtml()`. If the executor determines this is straightforward, implement it:

   ```typescript

   let highlighter: Awaited<ReturnType<typeof createHighlighter>> | null = null;

   async function getHighlighter() {
     if (!highlighter) {
       highlighter = await createHighlighter({
         themes: ['github-light', 'github-dark'],
         langs: ['java', 'javascript', 'typescript', 'sql', 'bash', 'yaml', 'json', 'dockerfile'],
       });
     }
     return highlighter;
   }
   ```

   Then use a custom `marked` renderer that intercepts code blocks and runs them through Shiki. If this adds excessive complexity, defer to Phase 2 and use basic `<pre><code>` styling with CSS.

1. Create `src/pages/[locale]/[section]/index.astro` -- section page with flashcard list:
   ```astro
   ---
   import { getCollection } from 'astro:content';
   import BaseLayout from '../../../layouts/BaseLayout.astro';
   import Header from '../../../components/Header.astro';
   import FlashcardList from '../../../components/FlashcardList.astro';
   import { t, localePath, type Locale, locales } from '../../../i18n/utils';
   import { getSectionMeta, getAllSectionSlugs } from '../../../i18n/sections';

   export async function getStaticPaths() {
     const sectionSlugs = getAllSectionSlugs();

     return locales.flatMap(locale =>
       sectionSlugs.map(section => ({
         params: { locale, section },
       }))
     );
   }

   const { locale, section } = Astro.params as { locale: Locale; section: string };
   const meta = getSectionMeta(section, locale);

   if (!meta) {
     return Astro.redirect(localePath(locale));
   }

   const allQuestions = await getCollection('questions');
   const sectionQuestions = allQuestions
     .filter(q => q.data.section === section)
     .sort((a, b) => a.data.order - b.data.order);

   // Map questions to the format Flashcard expects, selecting the correct locale
   const questionKey = `${locale}_question` as const;
   const answerKey = `${locale}_answer` as const;

   const questions = sectionQuestions.map(q => {
     // Extract slug from content entry ID (e.g., "qa/what-is-testing" -> "what-is-testing")
     const slug = q.id.split('/').pop() || q.id;
     return {
       slug,
       question: (q.data as any)[questionKey] as string,
       answer: (q.data as any)[answerKey] as string,
     };
   });
   ---
   <BaseLayout
     title={`${meta.name} - ${t(locale, 'site.title')}`}
     description={meta.description}
     locale={locale}
   >
     <Header locale={locale} />
     <main class="mx-auto max-w-4xl px-4 py-8 sm:px-6 sm:py-12">
       <div class="mb-8">
         <a href={localePath(locale)}
            class="inline-flex items-center gap-1 text-sm text-blue-600 hover:text-blue-800 mb-4 transition-colors">
           <svg class="w-4 h-4" fill="none" stroke="currentColor" viewBox="0 0 24 24">
             <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M15 19l-7-7 7-7" />
           </svg>
           {t(locale, 'section.back')}
         </a>
         <div class="flex items-center gap-3">
           <span class="text-3xl" role="img" aria-hidden="true">{meta.icon}</span>
           <div>
             <h1 class="text-2xl font-bold text-gray-900 sm:text-3xl">{meta.name}</h1>
             <p class="text-gray-600 text-sm mt-1">
               {questions.length} {t(locale, 'section.questions')}
             </p>
           </div>
         </div>
       </div>
       <FlashcardList questions={questions} />
     </main>

     <!-- Anchor link handler: opens and scrolls to linked question -->
     <script>
       function handleAnchorLink() {
         const hash = window.location.hash;
         if (!hash) return;

         const target = document.getElementById(hash.slice(1));
         if (!target) return;

         // Open the details element if it's a flashcard
         if (target.tagName === 'DETAILS') {
           (target as HTMLDetailsElement).open = true;
         }

         // Scroll to target with a short delay to allow details to open
         setTimeout(() => {
           target.scrollIntoView({ behavior: 'smooth', block: 'start' });
           target.classList.add('highlight');
           setTimeout(() => target.classList.remove('highlight'), 2000);
         }, 100);
       }

       // Handle initial page load
       handleAnchorLink();

       // Handle hash changes (clicking anchor links on the same page)
       window.addEventListener('hashchange', handleAnchorLink);
     </script>
   </BaseLayout>
   ```

   Key implementation details:
   - `getStaticPaths()` generates paths for all locale+section combinations
   - Questions are filtered by section and sorted by order field
   - Locale-appropriate question and answer fields are selected dynamically using `${locale}_question` / `${locale}_answer` key construction
   - The slug is extracted from the content entry ID (glob loader creates IDs like "qa/what-is-testing", we need just "what-is-testing" for the anchor)
   - Back link uses localePath() to navigate to home
   - Anchor link script runs on DOMContentLoaded and on hashchange
   - Script opens the matching `<details>` element and scrolls to it with highlight animation
   - The highlight CSS class is defined in Flashcard.astro (blue box-shadow ring)

   NOTE on type safety: The `(q.data as any)[questionKey]` cast is needed because TypeScript cannot narrow the dynamic key. This is a known trade-off of the flat field naming. An alternative is to use a helper function that properly types the access, but the `as any` is pragmatic for 4 well-known fields.
