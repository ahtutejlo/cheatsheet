en_question: "What is codegen in Playwright and when is it appropriate to use?"
ua_answer: |
  **Codegen** -- це вбудований інструмент Playwright для генерації тестового коду через запис дій користувача у браузері. Він запускається командою `npx playwright codegen` і відкриває браузер з інспектором, який записує кожну дію як TypeScript-код.

  Codegen автоматично обирає найкращі локатори (getByRole, getByText, getByLabel) та генерує assertions. Це корисний інструмент для швидкого прототипування тестів та вивчення API Playwright.

  ```typescript
  // Generated by codegen (example output):
  import { test, expect } from '@playwright/test';

  test('test', async ({ page }) => {
    await page.goto('https://example.com/');
    await page.getByRole('link', { name: 'Products' }).click();
    await page.getByRole('button', { name: 'Add to cart' }).click();
    await expect(page.getByText('Item added')).toBeVisible();
  });
  ```

  **Коли доречно використовувати:**
  - Швидке створення скелету тесту для нової функціональності
  - Вивчення API Playwright для початківців
  - Дослідження складних UI-взаємодій (drag-and-drop, multi-step forms)
  - Генерація правильних локаторів для незнайомого UI

  **Коли НЕ доречно:**
  - Генерований код не є production-ready -- його потрібно рефакторити
  - Codegen не створює Page Object Model -- структуру потрібно додати вручну
  - Згенеровані тести часто містять зайві кроки та не мають належних assertions
  - Codegen не обробляє складні сценарії (умовна логіка, цикли, тестові дані)

  ```typescript
  // Running codegen with specific options:
  // npx playwright codegen --target javascript https://example.com
  // npx playwright codegen --device "iPhone 14"
  // npx playwright codegen --save-storage=auth.json
  ```

  Codegen -- це стартова точка, а не кінцевий результат. Використовуйте його для швидкого початку, але завжди рефакторіть згенерований код у структурований, підтримуваний тест.
en_answer: |
  **Codegen** is Playwright's built-in tool for generating test code by recording user actions in the browser. It launches with `npx playwright codegen` and opens a browser with an inspector that records every action as TypeScript code.

  Codegen automatically selects the best locators (getByRole, getByText, getByLabel) and generates assertions. It is a useful tool for rapid test prototyping and learning the Playwright API.

  ```typescript
  // Generated by codegen (example output):
  import { test, expect } from '@playwright/test';

  test('test', async ({ page }) => {
    await page.goto('https://example.com/');
    await page.getByRole('link', { name: 'Products' }).click();
    await page.getByRole('button', { name: 'Add to cart' }).click();
    await expect(page.getByText('Item added')).toBeVisible();
  });
  ```

  **When appropriate to use:**
  - Rapid skeleton creation for new feature tests
  - Learning Playwright API for beginners
  - Exploring complex UI interactions (drag-and-drop, multi-step forms)
  - Generating correct locators for unfamiliar UI

  **When NOT appropriate:**
  - Generated code is not production-ready -- it needs refactoring
  - Codegen does not create Page Object Model -- structure must be added manually
  - Generated tests often contain redundant steps and lack proper assertions
  - Codegen does not handle complex scenarios (conditional logic, loops, test data)

  ```typescript
  // Running codegen with specific options:
  // npx playwright codegen --target javascript https://example.com
  // npx playwright codegen --device "iPhone 14"
  // npx playwright codegen --save-storage=auth.json
  ```

  Codegen is a starting point, not a final result. Use it for a quick start, but always refactor generated code into a structured, maintainable test.
