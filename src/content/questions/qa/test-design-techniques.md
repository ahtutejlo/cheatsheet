en_question: "What test design techniques do you know?"
ua_answer: |
  Техніки тест-дизайну -- це методи створення тестових випадків, які допомагають максимізувати покриття при мінімальній кількості тестів.

  **Основні техніки:**
  - **Еквівалентне розбиття (Equivalence Partitioning)** -- розподіл вхідних даних на класи еквівалентності, де кожен клас обробляється однаково
  - **Аналіз граничних значень (Boundary Value Analysis)** -- тестування значень на межах діапазонів
  - **Таблиця прийняття рішень (Decision Table)** -- таблиця комбінацій умов та очікуваних результатів
  - **Діаграма переходів станів (State Transition)** -- тестування переходів між станами системи
  - **Попарне тестування (Pairwise Testing)** -- комбінування параметрів попарно для зменшення кількості тестів

  **Техніки, засновані на досвіді:**
  - **Exploratory Testing** -- дослідницьке тестування без заздалегідь визначених кейсів
  - **Error Guessing** -- вгадування помилок на основі досвіду тестувальника
  - **Checklist-based Testing** -- тестування за чек-листом

  Вибір техніки залежить від типу системи, доступного часу та рівня ризику.
en_answer: |
  Test design techniques are methods of creating test cases that help maximize coverage with a minimum number of tests.

  **Main techniques:**
  - **Equivalence Partitioning** -- dividing input data into equivalence classes where each class is processed the same way
  - **Boundary Value Analysis** -- testing values at the boundaries of ranges
  - **Decision Table** -- a table of condition combinations and expected results
  - **State Transition Diagram** -- testing transitions between system states
  - **Pairwise Testing** -- combining parameters in pairs to reduce the number of tests

  **Experience-based techniques:**
  - **Exploratory Testing** -- investigative testing without predefined cases
  - **Error Guessing** -- guessing errors based on tester experience
  - **Checklist-based Testing** -- testing against a checklist

  The choice of technique depends on the system type, available time, and risk level.
