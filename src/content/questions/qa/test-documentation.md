ua_question: "Які документи створює тестувальник?"
en_question: "What documents does a tester create?"
ua_answer: |
  Тестувальник створює різні документи залежно від процесу, методології та потреб проекту.

  **Основні документи:**
  - **Тест-план (Test Plan)** -- стратегія тестування, обсяг, ресурси, графік
  - **Тест-кейс (Test Case)** -- детальний сценарій з кроками та очікуваним результатом
  - **Чек-лист (Checklist)** -- спрощений список перевірок без деталізації кроків
  - **Баг-репорт (Bug Report)** -- опис знайденого дефекту

  **Додаткові документи:**
  - **Тест-сюіт (Test Suite)** -- набір згрупованих тест-кейсів
  - **Матриця трасування (Traceability Matrix)** -- зв'язок вимог з тест-кейсами
  - **Звіт про тестування (Test Report)** -- підсумки тестування: скільки тестів пройдено, скільки багів знайдено
  - **Тестова стратегія (Test Strategy)** -- документ рівня організації, описує підхід до тестування

  **Структура баг-репорту:**
  - Заголовок, Severity, Priority
  - Кроки відтворення (Steps to Reproduce)
  - Очікуваний результат (Expected Result)
  - Фактичний результат (Actual Result)
  - Середовище (Environment)
  - Скріншоти або логи
en_answer: |
  A tester creates various documents depending on the process, methodology, and project needs.

  **Main documents:**
  - **Test Plan** -- testing strategy, scope, resources, schedule
  - **Test Case** -- detailed scenario with steps and expected results
  - **Checklist** -- simplified list of checks without detailed steps
  - **Bug Report** -- description of a found defect

  **Additional documents:**
  - **Test Suite** -- a set of grouped test cases
  - **Traceability Matrix** -- linking requirements to test cases
  - **Test Report** -- testing summary: how many tests passed, how many bugs found
  - **Test Strategy** -- organization-level document describing the testing approach

  **Bug report structure:**
  - Title, Severity, Priority
  - Steps to Reproduce
  - Expected Result
  - Actual Result
  - Environment
  - Screenshots or logs
