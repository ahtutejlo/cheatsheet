en_question: "What is the difference between static and dynamic testing?"
ua_answer: |
  **Статичне тестування** -- це перевірка програмного забезпечення без виконання коду. Воно включає аналіз документації, коду та артефактів.

  Приклади статичного тестування:
  - **Рев'ю коду (Code Review)** -- перевірка коду іншими розробниками
  - **Статичний аналіз** -- використання інструментів (SonarQube, ESLint, Checkstyle)
  - **Рев'ю вимог** -- перевірка специфікацій на повноту та несуперечливість
  - **Walkthrough** -- автор проводить команду через документ або код

  **Динамічне тестування** -- це перевірка програмного забезпечення шляхом виконання коду. Тестувальник взаємодіє з працюючою системою.

  Приклади динамічного тестування:
  - **Unit-тести** -- тестування окремих модулів
  - **Інтеграційне тестування** -- перевірка взаємодії модулів
  - **Функціональне тестування** -- перевірка відповідності вимогам
  - **Навантажувальне тестування** -- перевірка під навантаженням

  **Ключова різниця:** статичне тестування знаходить дефекти раніше та дешевше, динамічне -- перевіряє реальну поведінку системи.
en_answer: |
  **Static testing** is software verification without executing code. It includes analysis of documentation, code, and artifacts.

  Examples of static testing:
  - **Code Review** -- code verification by other developers
  - **Static Analysis** -- using tools (SonarQube, ESLint, Checkstyle)
  - **Requirements Review** -- checking specifications for completeness and consistency
  - **Walkthrough** -- author guides the team through a document or code

  **Dynamic testing** is software verification by executing code. The tester interacts with a running system.

  Examples of dynamic testing:
  - **Unit tests** -- testing individual modules
  - **Integration testing** -- verifying module interactions
  - **Functional testing** -- checking compliance with requirements
  - **Load testing** -- testing under load

  **Key difference:** static testing finds defects earlier and cheaper, dynamic testing verifies the actual behavior of the system.
