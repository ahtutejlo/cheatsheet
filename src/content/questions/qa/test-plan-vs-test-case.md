ua_question: "Яка різниця між тест-планом і тест-кейсом?"
en_question: "What is the difference between a test plan and a test case?"
ua_answer: |
  **Тест-план (Test Plan)** -- це документ високого рівня, який описує стратегію, обсяг, ресурси та графік тестування для проекту або релізу.

  Тест-план зазвичай включає:
  - Обсяг тестування (що тестуємо і що ні)
  - Типи тестування (функціональне, регресійне, навантажувальне)
  - Ресурси та відповідальності
  - Критерії входу та виходу
  - Ризики та план їх мітигації

  **Тест-кейс (Test Case)** -- це детальний опис окремого тестового сценарію з конкретними кроками, вхідними даними та очікуваними результатами.

  Тест-кейс включає:
  - Передумови (preconditions)
  - Покрокові дії (steps)
  - Очікуваний результат (expected result)
  - Фактичний результат (actual result)
  - Статус (pass/fail)

  **Головна різниця:** тест-план відповідає на питання "що і як тестувати на рівні проекту", а тест-кейс -- "як перевірити конкретну функцію крок за кроком".
en_answer: |
  **Test Plan** is a high-level document that describes the strategy, scope, resources, and schedule for testing a project or release.

  A test plan typically includes:
  - Test scope (what to test and what not)
  - Types of testing (functional, regression, load)
  - Resources and responsibilities
  - Entry and exit criteria
  - Risks and mitigation plan

  **Test Case** is a detailed description of an individual test scenario with specific steps, input data, and expected results.

  A test case includes:
  - Preconditions
  - Step-by-step actions
  - Expected result
  - Actual result
  - Status (pass/fail)

  **Key difference:** a test plan answers "what and how to test at the project level", while a test case answers "how to verify a specific function step by step".
