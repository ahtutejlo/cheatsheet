en_question: "What is the difference between verification and validation?"
ua_answer: |
  **Верифікація (Verification)** -- це процес перевірки того, чи продукт створюється правильно, відповідно до специфікацій та стандартів. Відповідає на питання: "Чи ми робимо продукт правильно?"

  Приклади верифікації:
  - Рев'ю вимог
  - Рев'ю дизайну
  - Рев'ю коду
  - Статичний аналіз
  - Інспекції (inspections)

  **Валідація (Validation)** -- це процес перевірки того, чи створений продукт відповідає потребам та очікуванням кінцевого користувача. Відповідає на питання: "Чи ми робимо правильний продукт?"

  Приклади валідації:
  - Функціональне тестування
  - User Acceptance Testing (UAT)
  - Beta-тестування
  - Демонстрації замовнику

  **Ключова відмінність:**
  - Верифікація -- **статичний** процес (без виконання коду)
  - Валідація -- **динамічний** процес (з виконанням коду)
  - Верифікація порівнює з **специфікацією**, валідація -- з **потребами користувача**
en_answer: |
  **Verification** is the process of checking whether the product is being built correctly, according to specifications and standards. It answers the question: "Are we building the product right?"

  Examples of verification:
  - Requirements review
  - Design review
  - Code review
  - Static analysis
  - Inspections

  **Validation** is the process of checking whether the built product meets the needs and expectations of the end user. It answers the question: "Are we building the right product?"

  Examples of validation:
  - Functional testing
  - User Acceptance Testing (UAT)
  - Beta testing
  - Customer demonstrations

  **Key difference:**
  - Verification is a **static** process (without code execution)
  - Validation is a **dynamic** process (with code execution)
  - Verification compares against **specification**, validation -- against **user needs**
