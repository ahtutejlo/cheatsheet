en_question: "What role does hashing play in blockchain?"
ua_answer: |
  **Хешування** -- це основа безпеки та цілісності блокчейну. Хеш-функція перетворює вхідні дані будь-якого розміру у рядок фіксованої довжини (хеш), який є унікальним для кожного набору даних.

  Роль хешування у блокчейні:
  - **Зв'язування блоків** -- кожний блок містить хеш попереднього блоку, утворюючи незмінний ланцюжок
  - **Цілісність даних** -- будь-яка зміна даних у блоці змінює його хеш та хеші всіх наступних блоків
  - **Proof of Work** -- майнери шукають nonce, при якому хеш блоку починається з певної кількості нулів
  - **Адреси гаманців** -- публічні адреси генеруються шляхом хешування публічного ключа
  - **Дерево Меркла** -- хеші транзакцій організуються у деревоподібну структуру для ефективної верифікації

  Найпоширеніша хеш-функція у блокчейні -- **SHA-256** (використовується в Bitcoin). Ethereum використовує **Keccak-256**.

  Властивості криптографічної хеш-функції: **детермінізм**, **швидкість обчислення**, **стійкість до колізій**, **незворотність**, **лавинний ефект** (мала зміна вхідних даних повністю змінює хеш).
en_answer: |
  **Hashing** is the foundation of blockchain security and integrity. A hash function converts input data of any size into a fixed-length string (hash) that is unique for each dataset.

  The role of hashing in blockchain:
  - **Block linking** -- each block contains the hash of the previous block, forming an immutable chain
  - **Data integrity** -- any change in block data changes its hash and the hashes of all subsequent blocks
  - **Proof of Work** -- miners search for a nonce that produces a block hash starting with a certain number of zeros
  - **Wallet addresses** -- public addresses are generated by hashing the public key
  - **Merkle tree** -- transaction hashes are organized into a tree structure for efficient verification

  The most common hash function in blockchain is **SHA-256** (used in Bitcoin). Ethereum uses **Keccak-256**.

  Properties of a cryptographic hash function: **determinism**, **computational speed**, **collision resistance**, **irreversibility**, **avalanche effect** (a small change in input completely changes the hash).
